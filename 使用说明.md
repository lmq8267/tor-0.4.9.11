# Tor 0.4.9.11 使用说明

> **安全公告**：Tor 0.4.9.11 是一个安全修复版本，修复了包括洋葱服务中间人攻击在内的多个高危安全问题。强烈建议尽快升级。

---

## 项目简介

本项目是 [Tor](https://www.torproject.org/)（第二代洋葱路由器）v0.4.9.11 的交叉静态编译版本，使用 musl libc 实现完全静态链接，无任何动态库依赖，可直接在目标系统上运行。

### 支持的平台

| 平台 | 架构 | 说明 |
|------|------|------|
| apple-darwin | x86_64 + arm64 Universal | macOS（Intel / Apple Silicon 通用二进制） |
| x86_64-pc-windows | x86_64 | Windows 64 位 |
| i686-pc-windows | i686 | Windows 32 位 |
| x86_64-linux-musl | x86_64 | Linux 64 位 |
| i686-linux-musl | i686 | Linux 32 位 |
| aarch64-linux-musl | AArch64 | Linux ARM64（树莓派4、手机等） |
| armv7-linux-musleabihf | ARMv7 硬浮点 | Linux ARM（树莓派2/3等） |
| armv7-linux-musleabi | ARMv7 软浮点 | Linux ARM |
| arm-linux-musleabihf | ARM 硬浮点 | Linux ARM（旧设备） |
| arm-linux-musleabi | ARM 软浮点 | Linux ARM（旧设备） |
| mipsel-linux-musl | MIPS 小端 | Linux MIPSEL（路由器等） |
| mips-linux-musl | MIPS 大端 | Linux MIPS（路由器等 |
| mips64-linux-musl | MIPS64 | Linux 64 位 MIPS |
| s390x-linux-musl | s390x | Linux IBM 大型机 |

### 产物中包含的二进制文件

每个平台的压缩包中包含以下 4 个程序：

| 文件名 | 说明 |
|--------|------|
| `tor` | Tor 主守护进程，支持客户端、中继、网桥模式 |
| `tor-resolve` | 通过 Tor SOCKS 代理解析域名的工具 |
| `tor-gencert` | 目录权威证书生成工具 |
| `tor-print-ed-signing-cert` | 查看 Ed25519 签名证书过期时间的工具 |

### 产物中包含的配置和数据文件

构建产物目录 `bin/` 中还会包含以下辅助文件：

| 文件名 | 说明 |
|--------|------|
| `geoip` | IPv4 GeoIP 数据文件，可通过 `GeoIPFile` 指定 |
| `geoip6` | IPv6 GeoIP 数据文件，可通过 `GeoIPv6File` 指定 |
| `torrc.sample` | Tor 配置文件示例，由 `src/config/torrc.sample.in` 复制生成 |

便携使用时建议保持这些文件和 `tor` 程序在同一个目录中：

```text
bin/
  tor
  tor-resolve
  tor-gencert
  tor-print-ed-signing-cert
  geoip
  geoip6
  torrc.sample
  torrc
  data/
```

`torrc.sample` 只是示例文件。实际使用时复制一份为 `torrc`，按需修改后通过
`tor -f ./torrc` 指定。文件后缀不重要，Tor 按内容解析配置文件。

## tor — 主守护进程

### 简介

`tor` 是 Tor 网络的核心程序。默认以客户端模式运行，通过 Tor 网络转发 TCP 流量（网页浏览、FTP、SSH 等），使接收者、观察者甚至中继节点本身都难以追踪流量来源。配置 `ORPort` 后可作为中继节点为网络提供带宽。

### 使用语法

```
tor [选项 值]
```

### 命令行专属选项

以下选项仅在命令行中有效，不能写在配置文件（torrc）中：

| 选项 | 参数 | 说明 |
|------|------|------|
| `-h` `--help` | 无 | 显示简要帮助信息并退出 |
| `-f` `--torrc-file` | `<文件路径>` | 指定 torrc 配置文件路径。传 `-` 可从标准输入读取。默认值：`/etc/tor/torrc`，若不存在则 `$HOME/.torrc` |
| `--defaults-torrc` | `<文件路径>` | 指定包含默认选项值的文件，会被 torrc 和命令行覆盖。默认值：`/etc/tor/torrc-defaults` |
| `--allow-missing-torrc` | 无 | 当 `-f` 指定的配置文件不存在时不报错（需 defaults-torrc 可访问） |
| `--ignore-missing-torrc` | 无 | 配置文件不存在时当作空文件处理，不产生警告 |
| `--verify-config` | 无 | 验证 torrc 配置文件是否有效，验证后退出 |
| `--dump-config` | `short` 或 `full` | 将当前配置输出到标准输出。`short` 仅输出与默认值不同的选项；`full` 输出所有选项 |
| `--hash-password` | `<密码>` | 生成密码的哈希值，用于控制端口认证（HashedControlPassword） |
| `--list-fingerprint` | `rsa`（默认）或 `ed25519` | 生成密钥并输出节点昵称和指纹 |
| `--keygen` | 无 | 生成 ed25519 主身份密钥。若已存在主密钥则生成新的临时签名密钥和证书 |
| `--newpass` | 无 | 修改 ed25519 主身份密钥的加密密码（配合 `--keygen` 使用） |
| `--no-passphrase` | 无 | 不加密/不解密密钥（配合 `--keygen` 使用） |
| `--passphrase-fd` | `<文件描述符>` | 从指定的文件描述符读取密码短语（配合 `--keygen` 使用），若未指定则从终端读取 |
| `--master-key` | `<文件路径>` | 指定备用的主密钥文件路径（配合 `--keygen` 使用） |
| `--keygen-family` | `<基础文件名>` | 生成家族身份密钥对，写入 `<基础文件名>.secret_family_key` 和 `<基础文件名>.public_family_id` |
| `--key-expiration` | `sign`（可选） | 查看签名密钥证书的过期时间 |
| `--format` | `iso8601`（默认）或 `timestamp` | 搭配 `--key-expiration` 使用，设置输出格式。`iso8601` 输出 ISO-8601 格式，`timestamp` 输出 Unix 时间戳 |
| `--version` | 无 | 显示 Tor 版本号并退出 |
| `--quiet` | 无 | 完全抑制控制台日志输出 |
| `--hush` | 无 | 仅在控制台显示 WARNING 及以上级别的日志 |
| `--list-torrc-options` | 无 | 列出所有有效的 torrc 配置选项名并退出 |
| `--list-deprecated-options` | 无 | 列出所有已废弃的配置选项名并退出 |
| `--list-modules` | 无 | 列出编译时包含的可选模块（relay、dirauth、dircache、pow） |
| `--nt-service` | 无 | 以 Windows NT 服务方式运行（仅 Windows） |

### 使用示例

```bash
# 作为客户端启动，监听默认 9050 端口
tor

# 指定 SOCKS 端口
tor --SocksPort 9999

# 指定配置文件
tor -f /etc/tor/torrc

# 使用便携包 bin/ 目录中的配置示例和 GeoIP 数据文件
cd /path/to/bin
cp torrc.sample torrc
mkdir -p data
tor -f ./torrc --DataDirectory ./data --GeoIPFile ./geoip --GeoIPv6File ./geoip6

# 后台运行并记录日志
tor --RunAsDaemon 1 --Log "notice file /var/log/tor/notices.log"

# 作为中继节点运行
tor --ORPort 9001 --Nickname MyRelay --ContactInfo "your@email.com" --DataDirectory /var/lib/tor

# 启用网桥模式
tor --UseBridges 1 --ClientTransportPlugin "exec /usr/bin/obfs4proxy" --Bridge "obfs4 1.2.3.4:1234 ABCDEF..."

# 验证配置文件
tor --verify-config -f /etc/tor/torrc

# 生成控制端口密码哈希值
tor --hash-password "your-password"

# 生成 ed25519 密钥
tor --keygen

# 查看密钥过期时间
tor --key-expiration sign

# 列出所有可用的 torrc 选项
tor --list-torrc-options
```

## tor-resolve — 域名解析工具

### 简介

`tor-resolve` 通过 Tor SOCKS 代理执行 DNS 解析，将域名解析为 IP 地址。可进行正向解析（域名→IP）和反向解析（IP→域名），所有 DNS 查询均通过 Tor 网络转发，保护隐私。

### 使用语法

```
tor-resolve [-4|-5] [-v] [-x] [-p 端口] 主机名 [SOCKS主机[:SOCKS端口]]
```

### 命令行选项

| 选项 | 说明 |
|------|------|
| `-5` | 使用 SOCKS5 协议进行解析（**默认**） |
| `-4` | 使用 SOCKS4a 协议进行解析。不支持反向 DNS 查询 |
| `-v` | 详细输出模式，显示 DEBUG 级别日志到标准错误（默认仅显示 WARN 及以上） |
| `-x` | 反向 DNS 查询：根据 IPv4 地址获取 PTR 记录。仅支持 SOCKS5 协议，与 `-4` 同时使用会报错 |
| `-p 端口` | 指定 SOCKS 代理端口号（1-65535），覆盖默认端口 9050 和命令行中指定的端口 |
| `主机名` | （必填）要解析的域名 |
| `SOCKS主机[:SOCKS端口]` | （可选）SOCKS 代理地址和端口，默认 `127.0.0.1:9050` |
| `--version` | 显示 Tor 版本号并退出 |

### 使用示例

```bash
# 解析域名（使用默认 SOCKS5 代理 127.0.0.1:9050）
tor-resolve example.com

# 使用 SOCKS4a 协议解析
tor-resolve -4 example.com

# 反向解析（IP → 域名）
tor-resolve -x 1.2.3.4

# 指定代理端口
tor-resolve -p 9055 example.com

# 指定代理主机和端口
tor-resolve example.com 192.168.1.1:1080

# 详细输出模式
tor-resolve -v example.com

# 指定代理主机（使用默认端口）
tor-resolve example.com 192.168.1.1
```

## tor-gencert — 证书生成工具

### 简介

`tor-gencert` 为 Tor 目录权威节点生成签名密钥和证书。它创建 v3 目录密钥证书，将签名密钥与身份密钥绑定，包含交叉签名和自签名。**只有运行目录权威节点的运维人员才需要使用此工具。**

每个目录权威都有一个长期的**身份密钥**（应离线安全保存）和一个短期的**签名密钥**（在线使用，用于签署投票和共识文档）。

### 使用语法

```
tor-gencert [-h|--help] [-v] [-r|--reuse] [--create-identity-key]
        [-i 身份密钥文件] [-s 签名密钥文件] [-c 证书文件]
        [-m 有效月数] [-a 地址:端口] [--passphrase-fd 文件描述符]
```

### 命令行选项

| 选项 | 参数 | 说明 |
|------|------|------|
| `-h` `--help` | 无 | 显示帮助信息并退出 |
| `-v` | 无 | 详细输出模式 |
| `-r` `--reuse` | 无 | 复用已有的签名密钥，仅生成新证书。用于更新地址或有效期 |
| `--create-identity-key` | 无 | 生成新的身份密钥（3072 位 RSA）。仅在首次运行时使用 |
| `-i` | `<文件路径>` | 读取/写入身份密钥的文件路径。默认：`./authority_identity_key` |
| `-s` | `<文件路径>` | 写入签名密钥的文件路径。默认：`./authority_signing_key` |
| `-c` | `<文件路径>` | 写入证书的文件路径。默认：`./authority_certificate` |
| `-m` | `<月数>` | 证书有效月数，范围 0-24。默认：12 |
| `-a` | `<地址:端口>` | 设置证书中的 `dir-address` 字段（IPv4 地址和端口） |
| `--passphrase-fd` | `<文件描述符>` | 从指定的文件描述符读取身份密钥的密码短语（以换行符或 NUL 结尾）。默认从终端读取 |

### 安全说明

- 所有生成的文件权限为 `0400`（身份密钥）或 `0600`（签名密钥），密码保护使用 3DES 加密
- 身份密钥（`-i` 文件）应离线保存，**不要** 复制到 Tor 运行目录的 `keys/` 子目录中
- 仅需将签名密钥（`-s` 文件）和证书（`-c` 文件）复制到 Tor 的 `keys/` 子目录，然后向 Tor 发送 SIGHUP 信号使其重新加载
- 最小密码长度为 4 个字符
- `-a` 参数仅支持 IPv4 地址

### 使用示例

```bash
# 首次运行：生成身份密钥
tor-gencert --create-identity-key

# 生成证书（使用已有身份密钥，有效 12 个月）
tor-gencert -a 192.0.2.1:443

# 复用签名密钥生成新证书（更新地址或有效期）
tor-gencert -r -a 192.0.2.1:8080

# 指定证书有效期为 6 个月
tor-gencert -m 6 -a 192.0.2.1:443

# 指定所有文件路径
tor-gencert --create-identity-key \
  -i /secure/keys/identity_key \
  -s /secure/keys/signing_key \
  -c /secure/keys/certificate

# 从文件描述符读取密码
echo "my-password" | tor-gencert --create-identity-key --passphrase-fd 0
```

## tor-print-ed-signing-cert — Ed25519 签名证书查看工具

### 简介

`tor-print-ed-signing-cert` 是一个轻量级工具，用于读取 Ed25519 签名证书文件并以三种格式输出其过期时间。主要供 Tor 中继节点运维人员检查证书有效期。

### 使用语法

```
tor-print-ed-signing-cert <ed25519签名证书文件路径>
```

### 参数说明

| 参数 | 说明 |
|------|------|
| `<ed25519签名证书文件路径>` | （必填）包含 ed25519 签名证书的文件路径 |

### 输出格式

程序会以三种格式输出证书过期时间：

1. **本地时间格式**：如 `Expires at: Wed Jun 21 12:34:56 2023`（根据当前时区）
2. **RFC 1123 格式**：如 `RFC 1123 timestamp: Thu, 22 Jun 2023 04:34:56 GMT`（GMT 时区）
3. **Unix 时间戳格式**：如 `UNIX timestamp: 1687418096`（自 1970-01-01 起的秒数）

### 使用示例

```bash
# 查看 Ed25519 签名证书的过期时间
tor-print-ed-signing-cert /var/lib/tor/keys/ed25519_signing_cert

# 查看自定义数据目录中的证书
tor-print-ed-signing-cert ~/.tor/keys/ed25519_signing_cert
```

## 快速开始（客户端模式）

以下是在 Linux 上以客户端模式快速启动 Tor 的步骤。假设已经进入产物
`bin/` 目录，且 `geoip`、`geoip6`、`torrc.sample` 与 `tor` 在同一目录：

```bash
# 1. 创建数据目录
mkdir -p data

# 2. 从示例复制实际配置文件
cp torrc.sample torrc

# 3. 写入或追加便携运行所需的基本配置
cat >> torrc << 'EOF'
SocksPort 9050
DataDirectory ./data
GeoIPFile ./geoip
GeoIPv6File ./geoip6
Log notice file ./data/notices.log
EOF

# 4. 启动 Tor
./tor -f ./torrc

# 5. 等待连接建立后，通过 Tor 代理上网
# 例如配合 curl 使用：
curl --socks5-hostname 127.0.0.1:9050 https://check.torproject.org/api/ip
```

Windows 下同样可以把 `geoip`、`geoip6`、`torrc.sample` 放在 `tor.exe`
所在目录，复制出 `torrc` 后在配置中使用：

```conf
DataDirectory .\data
GeoIPFile .\geoip
GeoIPv6File .\geoip6
```

然后执行：

```powershell
.\tor.exe -f .\torrc
```

## 常见问题

**Q：启动后提示 "Problem bootstrapping" 怎么办？**

A：检查网络连接是否正常，是否需要代理才能访问外网。如需上游代理，在 torrc 中添加：
```
HTTPSProxy 127.0.0.1:8080
```

**Q：如何验证 Tor 是否正常工作？**

A：启动后访问 `https://check.torproject.org`，或使用 `tor-resolve` 工具测试：
```bash
./tor-resolve check.torproject.org
```

**Q：客户端模式和中继模式有什么区别？**

A：客户端模式（默认）仅通过 Tor 网络转发自身流量；中继模式配置 `ORPort` 后会转发其他用户的流量，为 Tor 网络提供带宽。

**Q：如何配置网桥？**

A：在 torrc 中添加：
```
UseBridges 1
ClientTransportPlugin obfs4 exec /usr/bin/obfs4proxy
Bridge obfs4 [IP]:[端口] [指纹] cert=... iat-mode=...
```

**Q：如何查看 Tor 的版本号？**

A：`tor --version`

**Q：Windows 上如何以服务方式运行？**

A：`tor --service install --options -f C:\path\to\torrc`，然后通过 `--service start/stop/remove` 管理服务。
